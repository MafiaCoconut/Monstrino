  5. CONTENT RULES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Context          → Explain WHY a decision was needed, not what was decided.
                     Include constraints and what failure would look like.

  Options          → Minimum 2, typically 3. Mark chosen option with ✅.
                     Keep each option brief — pros and cons as bullets.

  Decision         → Always a blockquote (> ...).
                     Start with "We choose / We adopt / We reject".
                     State the primary rationale, not just the choice.

  Consequences     → Both Positive AND Negative are required.
                     Bullets only. Be honest about drawbacks.

  Related Decisions → Optional section. Use only if genuinely related.
                      Same directory:  [ADR-X-NNN](./adr-x-nnn.md)
                      Cross directory: [ADR-X-NNN](../category/adr-x-nnn.md)
                      Never link to the ADR itself (self-reference).
