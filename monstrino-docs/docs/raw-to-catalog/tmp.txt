=
Нужно выделить что из текущего описания мы не можем написать алгоритм который
будет в 100% случаев получать корректные данные
в одном случае он реально вернет например characters,
а в другом вернет другие рандомные слова
= 

Данные в таком формате собирает catalog-collector сервис и сохраняет их в бд
в схеме ingest для сырых спарсенных данных

Дальше за дело берется сервис `catalog-data-enricher` который с помощью AI
дозаполнит эти данные из описания релиза

Он возьмет эту модель данных, возьмет поле mpn, запросит у сервиса `catalog-api-service`
информацию существует ли такой релиз, если он существует, тогда `catalog-data-enricher`
попробует из новополученных данных получить недостающие данные
если такого релиза в бд нет, тогда `catalog-data-enricher` начнет заполнять недостающие
данные из текущей модели данных

пример запроса в `catalog-api-service`

```json
{
  "query": {
    "filters": { "mpn": "JDR52"},
    "page": { "limit": 10, "offset": 0 },
    "include": {"id": true, "mpn": true}
  },
  "context": { "locale": "en" }
}
```

после получения ответа `catalog-data-enricher` начинает идти по каждому атрибуту
и проверять если в нем данные

возьмем для примера поле `characters`

сервис увидит что это поле = None

после чего он вызовет через AiOrchestartorClient API запрос к сервису `ai-orchestrator`
и в качестве payload передаст контекстные поля

```json

{
    "scenario_name": "enrich-release-characters",
    "payload":{
        "mpn": "",
        "title": "",
        "description": "",
        "content_description": "",
    }
}

```

`catalog-data-enricher` выполнит api запросы для всех пропущенных данных в `ai-orchestrator`
и закончит выполнение, поставив processing_state у этого релиза в `waiting-for-enrichment`

когда `ai-orchestrator` закончит, он сам вызовет через api запрос в `catalog-data-enricher`
что поступили новые данные по такому-то полю

внутри сервиса `ai-orchestrator` уже прописаны промпт, системный промпт, алгоритм
вывыдения characters из описания

после вызова сценария, UseCase передаст эти данные в LLM запущенную через
сервис `ollama`

Модель проанализирует эти данные и вернет соотвествующую модель данных
 
```json
{
```

и он уже сохранит эти данные в бд

после того как по всем данным будет получен ответ от `ai-orchestrator`
`catalog-data-enricher` переведет processing_state в статус `ready-for-import`
