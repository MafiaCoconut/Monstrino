# Architectural Patterns

Monstrino follows several established software architecture patterns
to ensure maintainability, scalability, and clear separation of concerns.

These patterns are applied consistently across all services.

## Patterns used in Monstrino

- Clean Architecture
- Repository Pattern
- Unit of Work
- Dependency Inversion
- Domain-Driven Design principles
- Layered Architecture

Each pattern plays a specific role in structuring the platform.

## Goals

Using these patterns allows Monstrino to:

- isolate business logic from infrastructure
- maintain clear boundaries between layers
- simplify testing
- support long-term scalability
- reduce coupling between components
